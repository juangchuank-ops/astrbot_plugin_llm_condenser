# LLM 回复简化器 (LLM Condenser)

自动简化大语言模型（LLM）的回复内容，去除冗余表达，同时严格保留核心信息和情感色彩。

## 功能特性

- **自动拦截**：通过 `on_llm_response` 钩子，在 LLM 回复发送给用户前自动处理
- **智能去冗余**：移除填充词、自我指代、重复表达等 30+ 种冗余模式
- **情感保留**：识别并保留感叹号、表情符号、情绪关键词等情感标记
- **内容去重**：检测重复句子并合并，避免信息重复
- **长度控制**：支持最大字符数限制，超长文本在句子边界处优雅截断
- **中英双语**：同时支持中文和英文回复的简化处理

## 安装方法

1. 将 `astrbot_plugin_llm_condenser` 文件夹复制到 AstrBot 的 `addons/plugins` 目录下
2. 重启 AstrBot 或在 Web UI 中加载插件

```
addons/plugins/
└── astrbot_plugin_llm_condenser/
    ├── _conf_schema.json    # 配置项定义（Web UI 展示）
    ├── main.py              # 插件入口
    └── metadata.yaml        # 插件元数据
```

## 配置项

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `enabled` | bool | `true` | 是否启用简化功能 |
| `max_length` | int | `800` | 简化后的最大字符数（100-5000） |
| `debug` | bool | `false` | 是否在控制台输出简化日志 |

配置可在 AstrBot Web UI 的插件管理页面中直接修改。

## 简化规则

### 第一层：填充词移除

自动识别并移除非信息性的表达：

| 类别 | 示例 |
|------|------|
| 自我指代 | 作为AI助手、我认为、在我看来、据我所知 |
| 过渡短语 | 总的说来、简单来说、换句话说、当然 |
| 口语化填充 | 嗯、呃、那么、接下来 |
| 英文填充 | As an AI assistant, I think, In my opinion, To be honest |

### 第二层：表达收紧

合并冗余的语气强调和程度副词：

```
非常非常 → 非常
真的真的 → 真的
very very → very
really really → really
```

### 第三层：标点规范化

- 合并连续重复的标点符号（`！！！` → `！`）
- 规范化多余空白字符

### 第四层：语义去重

基于关键词签名检测内容重复的句子，仅保留首次出现，除非该句包含情绪表达。

### 第五层：长度截断

当简化后的文本仍超过 `max_length` 时，在最近的句末标点处截断，避免破坏语义完整性。

## 处理示例

### 示例 1：中文冗余
```
输入：作为一个AI助手，我认为这个方案非常非常不错，总的来说值得一试！
输出：这个方案非常不错，值得一试！
```

### 示例 2：英文冗余
```
输入：As an AI assistant, I think this is a really really great idea. To be honest, it's worth trying!
输出：This is a really great idea. It's worth trying!
```

### 示例 3：情感保留
```
输入：据我所知，天哪！太棒了！！太好了！真的真的非常开心！
输出：天哪！太棒了！太好了！真的非常开心！
```
> 注意：多个感叹句因情感标记不同而被保留。

### 示例 4：长度截断
```
输入：一篇超过800字符的详细技术说明...
输出：在800字符处最后一个句号处截断的版本...
```

## 技术架构

```
LLM 生成回复
    ↓
on_llm_response 钩子触发
    ↓
LLMCondenser._condense() 处理
    ├── 填充词正则移除
    ├── 表达收紧
    ├── 标点规范化
    ├── 语义去重（情绪感知）
    └── 长度安全截断
    ↓
修改 resp 对象文本
    ↓
AstrBot 将处理后的文本发送给用户
```

插件通过修改 `LLMResponse` 对象的响应文本字段实现"无感"介入，不影响 AstrBot 的其他功能。
